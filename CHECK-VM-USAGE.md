# check-vm.sh — инструкция

Скрипт проверяет подходит ли VM для использования как **relay-нода** для обхода белых списков мобильных операторов РФ.

---

## Зачем это нужно

Мобильные операторы (МТС, Мегафон, Tele2, Beeline, Yota) пропускают трафик **только к IP-адресам из своих белых списков**.
Туда попадают Yandex Cloud, VK Cloud, Aeza и другие крупные RU-облака.
Обычный VPS с российским IP — **не гарантирует** попадание в белый список.

Этот скрипт проверяет именно это.

---

## Требования

На VM должны быть установлены:

```bash
apt-get install -y curl jq netcat-openbsd iputils-ping
```

На Ubuntu 22.04 всё уже есть кроме `jq` — ставится одной командой.

---

## Запуск

### Минимальный (только проверка IP/ASN)

```bash
bash check-vm.sh
```

### Полный (+ проверка выхода к твоему VPS)

```bash
bash check-vm.sh 1.2.3.4
```

Где `1.2.3.4` — IP твоей ноды в Нидерландах или Финляндии.

### Если нет файла на VM — скачай и запусти сразу

```bash
# Только проверка IP/ASN
bash <(curl -fsSL https://raw.githubusercontent.com/kalitkin/relay-ru/main/check-vm.sh)

# С проверкой выхода к твоему VPS
bash <(curl -fsSL https://raw.githubusercontent.com/kalitkin/relay-ru/main/check-vm.sh) 1.2.3.4
```

### С подробным выводом (raw API ответы)

```bash
VERBOSE=1 bash check-vm.sh
```

---

## Что проверяет

| # | Проверка | Влияет на результат |
|---|----------|-------------------|
| 1 | Публичный IP определён | FAIL если нет интернета |
| 2 | Страна = RU | WARN если не RU (ASN важнее) |
| 3 | ASN провайдера в белом списке | PASS / WARN / FAIL |
| 4 | IP входит в известные CIDR | Только info |
| 5 | Интернет работает (ya.ru / vk.com) | FAIL если оба недоступны |
| 6 | Порт 443 свободен | WARN если занят |
| 7 | Пинг до ya.ru | Только info |
| 8 | TCP до твоего VPS (если передан) | WARN если недоступен |

---

## Результаты

### ✅ VM ПОДХОДИТ
Все критические проверки пройдены, ASN известный.
Можно ставить relay: `sudo bash setup-relay.sh`

### ⚠ ВОЗМОЖНО ПОДОЙДЁТ
Нет критических ошибок, но есть предупреждения.
Например, ASN не в списке, но страна RU и всё работает.
Попробуй поставить relay — скорее всего заработает.

### ✗ VM НЕ ПОДХОДИТ
Критические проверки провалены. Ищи другую VM.

## Где брать подходящий VM

| Провайдер | Сайт | Цена | Примечание |
|-----------|------|------|-----------|
| **Yandex Cloud** | cloud.yandex.ru | ~400 ₽/мес | Preemptible VM, лучший вариант |
| **Aeza RU** | aeza.net | от 200 ₽/мес | Есть RU-локация |
| **VK Cloud** | cloud.vk.com | ~300 ₽/мес | Mail.ru инфраструктура |
| **Selectel** | selectel.ru | от 500 ₽/мес | Надёжно |
| **Timeweb Cloud** | timeweb.cloud | от 200 ₽/мес | |

> Не использовать: Hetzner, DigitalOcean, AWS, OVH — их подсети массово заблокированы в РФ.

## Частые вопросы

**Q: Страна показывает не RU — это плохо?**
Нет. Некоторые API неточно определяют страну по IP. ASN важнее.

**Q: ASN не в списке, но VM в Yandex Cloud — что делать?**
Запусти с `VERBOSE=1` и посмотри raw ответ. Если `org` содержит "Yandex" или "YANDEX" — скорее всего подойдёт. Попробуй установить relay.

**Q: Порт 443 занят. Фатально?**
Нет. В `setup-relay.sh` укажи `RELAY_PORT=8443` или другой свободный порт.

**Q: Upstream VPS недоступен.**
Это нормально до тех пор, пока ты не добавил inbound в Marzban на VPS. После добавления повтори проверку.
